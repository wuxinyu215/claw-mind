## 🎯 推荐写法（用无单位数字）

```css
p {
  line-height: 1.6;   /* 相对当前 font-size，最灵活 */
}
```

**为什么推荐无单位？** 因为子元素会继承比例，不会因为 `font-size` 不同而错乱。

## 📝 其他写法对比

```css
line-height: 1.6;        /* ✅ 推荐：相对自身 font-size */
line-height: 24px;       /* 固定像素 */
line-height: 160%;       /* 计算后继承（子元素 font-size 不同时会出问题） */
line-height: 1.6em;      /* 类似百分比 */
```

## 🎨 不同场景的推荐值

| 场景 | 值 |
|---|---|
| **中文正文** | `1.6 - 1.8`（中文密度大，需要更松） |
| **英文正文** | `1.4 - 1.6` |
| **标题** | `1.2 - 1.4` |
| **按钮 / 单行** | `1` 或按 `height` 设 |
| **大段说明** | `1.7 - 2.0`（可读性更好） |

## ⚠️ 常见坑

- **行高算行内元素高度**：单行文字时，`height` 可以省略，用 `line-height` + `vertical-align: middle` 居中
- **不要用百分比/em**：子元素 font-size 不同时会算出奇怪的像素
- **中英文混排**：中文需要更大行间距（密度高）

## 🔍 最可能的原因（90% 的情况）

### 1. `line-height` 默认 ≈ 1.2

每个文字行都会因为 line-height 撑出额外空间。比如：

```html
<td>
  <div>有 5 行文字</div>
</td>
```

`div` 高度 100px，但里面 5 行文字 × line-height 1.2 × 16px 字号 = 96px 文字 + 上下各约 4px 多余空间。

**如果 td 里有 5 行文字，每行多 2px，累积就是 10px**，刚好对得上你说的 216 - 205.68 ≈ 10.32px。

### 2. 空白节点

```html
<td>
  <div>...</div>     <!-- 这个换行会被算成 1 个文字高度 -->
  <span>...</span>   <!-- 这里也是 -->
</td>
```

### 3. td 默认 padding 1px

```css
td { padding: 1px; }  /* 上下各 1px = 2px */
```

## 💡 最干净的方案

如果你想让 td 高度 = 内容高度：

```css
td {
  height: auto;          /* 不要硬定高度 */
  line-height: 1;        /* 子元素继承前可以重置 */
  padding: 0;
  font-size: 0;          /* 关键：消除空白节点 */
}

td > * {
  font-size: 16px;       /* 子元素单独设字号 */
}
```

或者更简单，**全部子元素写一行**，消除空白节点：

```html
<td><div>...</div><span>...</span></td>
```

## ⚠️ 一个反直觉的点

`td` 是 `display: table-cell`，**子元素的 `line-height` 会影响 td 总高度**——和普通 div 不一样。

也就是说即使你 `td { line-height: 0 }`，**子元素自己还是会撑开**。所以要把子元素的 line-height 也设了。
