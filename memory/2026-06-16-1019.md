## 几个细节

| 场景 | 处理方式 |
|---|---|
| radio 按钮 | 按 `name` 找 + `value` 匹配（不是按 id） |
| `disabled` / `readonly` 字段 | 跳过不写（避免报错） |
| 框架不响应？ | 手动派发了 `input` + `change` 事件 |
| **超过 maxlength** | 浏览器会**自动截断**，不会报错 |
| 找不到元素 | 收集到 `errors` 数组，最后 console.table 出来 |
| 特殊字符（name 里有引号） | 用 `CSS.escape()` 处理 |

## 关于 file:// 的坑

| 方式 | 能不能读本地文件 | 备注 |
|---|---|---|
| `fetch('file:///...')` | ❌ | Chrome 安全策略直接拒绝 |
| `fetch('http://...')` | ❌ | 本地文件没有 HTTP 地址 |
| `FileReader` + `<input type="file">` | ✅ | **唯一可行的浏览器内方式** |
| `GM_xmlhttpRequest`（油猴）| ❌ | 也受 file:// 限制 |
| 启个本地 HTTP 服务 | ✅ | 用 `python -m http.server` 起一个 |
